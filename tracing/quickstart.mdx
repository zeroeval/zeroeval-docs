ZeroEval provides comprehensive distributed tracing capabilities to help you monitor, debug, and optimize your AI applications. This guide introduces the core concepts and helps you get started.

Tracing allows you to monitor and debug your AI applications by capturing detailed information about each operation:

- **Spans**: Individual units of work (e.g., an API call, a function execution)
- **Traces**: Complete request flows composed of multiple spans
- **Sessions**: Groups of related traces (e.g., a user conversation)
- **Attributes**: Metadata attached to spans for filtering and analysis

## Patterns

### 1. Basic Setup

Both SDKs follow a similar initialization pattern:

<CodeGroup>
```python Python
import zeroeval as ze

# Use environment variable
ze.init()

# Or provide API key directly
ze.init(api_key="YOUR_API_KEY")
```

```typescript TypeScript
import * as ze from '@zeroeval/sdk';

// Use environment variable
ze.init();

// Or provide API key directly
ze.init({ apiKey: 'YOUR_API_KEY' });
```
</CodeGroup>

### 2. Creating Spans

Wrap any operation to create a span:

<CodeGroup>
```python Python
@ze.span(name="my_operation")
def my_function():
    return "result"
```

```typescript TypeScript
const result = await ze.withSpan(
  { name: 'my_operation' },
  () => "result"
);
```
</CodeGroup>

### 3. Error Handling

Errors are automatically captured in both SDKs:

<CodeGroup>
```python Python
@ze.span(name="risky_op")
def risky_operation():
    raise Exception("Error!")
    # Error automatically traced
```

```typescript TypeScript
ze.withSpan({ name: 'risky_op' }, () => {
  throw new Error('Error!');
  // Error automatically traced
});
```
</CodeGroup>

### 4. Custom Attributes

Add metadata for better filtering:

<CodeGroup>
```python Python
@ze.span(
  name="inference",
  attributes={
    "model": "gpt-4",
    "temperature": 0.7
  }
)
def generate():
    # Your code
```

```typescript TypeScript
ze.withSpan({
  name: 'inference',
  attributes: {
    model: 'gpt-4',
    temperature: 0.7
  }
}, () => {
  // Your code
});
```
</CodeGroup>

### 5. Manual I/O Control

Control what data is captured:

<CodeGroup>
```python Python
@ze.span(name="process")
def process_file(path):
    content = read_file(path)
    
    # Capture summary, not content
    span = ze.span.get_current()
    span.set_io(
        input_data=f"File: {path}",
        output_data="Complete"
    )
```

```typescript TypeScript
ze.withSpan({ name: 'process' }, () => {
  const content = readFile(path);
  
  // Capture summary, not content
  const span = ze.getCurrentSpan();
  span?.setIO(
    `File: ${path}`,
    'Complete'
  );
});
```
</CodeGroup>

## Configuration

Both SDKs support similar configuration options:

<CodeGroup>
```python Python
ze.init(
  api_key="YOUR_API_KEY",
  # Additional options
)

# Fine-tune tracer
from zeroeval.observability.tracer import tracer
tracer.configure(
  flush_interval=5.0,
  max_spans=200
)
```

```typescript TypeScript
ze.init({
  apiKey: 'YOUR_API_KEY',
  flushInterval: 5,
  maxSpans: 200,
  integrations: {
    openai: true,
    langchain: true
  }
});
```
</CodeGroup>
